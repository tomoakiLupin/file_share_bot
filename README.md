# 🎈 Discord 文件分享辅助机器人 (File Share Bot)

这是一个专门为 Discord 社区设计的“文件分享与访问控制”辅助机器人。通过使用 Discord 最新的组件框架 (Components V2) 和斜杠命令 (Slash Commands)，它可以在论坛频道 (Forum Channels) 中自动监测文件分享帖，并为用户提供一个优雅、安全、可控的文件分发中心。

---

## 🛠️ 核心功能与命令列表

机器人共提供了 **7 个核心斜杠命令**，覆盖了从发布到管理的完整生命周期：

### 📝 发布与分享

- **`/发布作品`**
  - **权限**: 仅限发帖人（楼主）或服务器管理员。
  - **功能**: 呼出一个可视化的模态框 (Modal)，让发布者填写文件直链（如蓝奏云、网盘链接）以及获取条件。提交后，机器人会在帖子内生成一个精美的“获取面板”。
- **`/获取作品`**
  - **权限**: 所有用户。
  - **功能**: 在包含分享面板的帖子里使用，一键获取该帖子内发布的最新文件。

### 📊 数据统计与查询

- **`/查询作品`**
  - **参数**: `file_id` (作品编号)
  - **权限**: **分级权限**。
    - 普通用户：拒绝访问。
    - 文件发布者：可以看到总下载次数，以及最近下载的**匿名时间点**。
    - 服务器管理员/服主：不仅能看到总次数，还能看到**具体是由哪些用户（精确到 Discord 具体的 UID）在什么时间**下载了该文件。
- **`/查询用户`**
  - **参数**: `user` (@某人)
  - **权限**: 仅限服务器管理员/服主。
  - **功能**: 输入一个特定的用户，查询该用户在服务器内**历史累积的总下载/获取次数**，以及该用户最近获取的具体文件 ID 列表和时间。

### ⚙️ 管理与设置

- **`/移除作品`**
  - **参数**: `file_id` (作品编号)
  - **权限**: 文件发布者 或 服务器管理员。
  - **功能**: 从数据库中彻底销毁该文件的记录与访问权限。
- **`/关闭自动提示`**
  - **功能**: 个人设置。当用户发布新帖子时，机器人默认会发送一条隐藏消息提示“是否需要发布作品”。如果觉得烦，可以使用此命令永久关闭该提示。
- **`/启用自动提示`**
  - **功能**: 重新开启发帖时的自动提示。

## � 部署与初始化配置 (Setup Guide)

如果你想把这个机器人部署到自己的服务器上，必须先为它配置好身份信息和所在的频道信息。

### 步骤 1：获取 Bot Token

1. 去 [Discord Developer Portal](https://discord.com/developers/applications) 新建一个应用 (Application)。
2. 在左侧菜单点击 `Bot`，找到 `Reset Token` 并复制生成的那串密文（这是机器人的唯一通行证）。
3. **⚠️ 极其重要**：在同一个 `Bot` 页面，往下拉找到 **Privileged Gateway Intents**，把三个开关都打开（特别是 `Message Content Intent` 和 `Server Members Intent`）。
4. 在机器人代码的根目录下，新建一个文本文档，命名为 `.env`，然后在里面写入：
   ```text
   DISCORD_TOKEN=在此处粘贴你的Bot密文
   ```

### 步骤 2：开启开发者模式获取 ID

在 Discord 客户端中，前往 **用户设置 (User Settings) > 高级设置 (Advanced)**，打开 **开发者模式 (Developer Mode)** 功能。只有打开这个，你才能复制频道和服务器的纯数字 ID。

### 步骤 3：修改配置文件 `bot_config.json`

打开 `data/config/bot_config.json` 文件进行绑定：

1. **服务器 ID (Guild ID)**：右击你想让机器人进驻的服务器头像，选择 `复制服务器 ID`。
   将其填入 `main_config.safety_setting.command_push_guildids` 这个数组里（斜杠命令只会在这个服务器生效）。
2. **日志频道 ID (Log Channel)**：新建一个小黑屋文字频道，右击选择 `复制频道 ID`。
   将其填入 `logger.log_channel_id` 中，机器人所有的启动成功、文件下载成功、错误排查信息都会在这里汇报。
3. **论坛主版块 ID (Forum Channels)**：右击你想让机器人**自动提示是否发布面板的那个主论坛版块**（不是里面的一个个帖子），选择 `复制频道 ID`。
   将其填入 `forum_rules.channels`。

配置完成后，运行 `npm start`，如果日志频道出现了 `[bot] 所有模块初始化完成，机器人就绪！` 则代表配置成功拉起！

## �🛡️ 防护措施与安全设计

为了防止服务器被滥用、被爬虫抓取，以及保护发布者的权益，本系统的核心设计包含了以下防护：

1.  **全局每日下载限制 (Rate Limit)**
    - 在 `bot_config.json` 中配置了 `daily_download_limit`。每个普通用户每天只能获取固定数量的新文件。
    - **智能防呆**: 如果一个用户在同一天内多次点击获取**同一个文件**，系统不会重复扣除他的每日额度。
2.  **细粒度权限隔离 (Role-Based Access Control)**
    - 无论是面板上的按钮，还是斜杠命令，全部自带权限校验。普通用户点击别人的“移除面板”按钮会直接报错。
    - 底层数据库的查询做到了“服主、发布者、访客”三者数据视角的严格隔离。
3.  **获取条件壁垒 (Requirements Check)**
    - 支持强制要求用户“必须点赞”、“必须完成验证码（预留接口）”、“必须同意免责声明”后才能看到真实的下载链接。
4.  **隐式直链分发**
    - 下载链接不会直接暴露在帖子中，而是通过 `[隐藏消息/Ephemeral Message]` 的方式单独发送给通过验证的用户，防止链接被未入群的外部爬虫直接刮取。

## 🧪 测试环境注意事项

在进行系统测试或二次开发时，请注意以下几点：

### 1. 详细操作日志记录 (测试专用 Logger)

系统目前内置了一个**高精度的动作监视器** (`utils/test_activity_logger.js`)，它会拦截并记录 Discord 客户端发来的**每一个**操作（包括谁点击了什么按钮、输入了什么命令、填了什么表单）。

- **如何查看？** 所有的记录都会按天生成独立的日志文件，保存在你项目根目录的 `logs/` 文件夹下（例如：`logs/activity_2024-03-08.log`）。你可以用任何文本编辑器（如记事本、VSCode）打开它查看具体的追踪记录。
- **如何关闭？** 当正式上线不需要这么密集的日志时，请前往 `bot/index.js`，找到 `this.client.on('interactionCreate')`，将其中的 `require('../utils/test_activity_logger').logInteraction(interaction);` 一行注释或删除即可。

### 2. 数据库状态清理

- 所有的文件数据、下载记录都存在 `data/shared_files.sqlite` 这个本地数据库文件里。
- 如果在测试过程中产生了很多垃圾数据，或者你想重置所有人的“每日下载次数”进行全新测试，你可以直接在关闭机器人的情况下，**删除或重命名 `shared_files.sqlite` 文件**。机器人下次启动时会自动生成一个干净的空数据库。

### 3. 组件 (Component V2) 兼容性

代码目前严格按照 Discord 组件 V2 的要求排列（例如 ActionRow 中只能包含同类型的按钮或选单，不能混搭）。如果在后续修改面板 UI 时遇到 `DiscordAPIError[50035]: Invalid Form Body`，请检查组件堆叠顺序或是否包含了非法字符（如已修复的乱码 Emoji）。
